Web APIs: работа с возможностями браузера

Browser API

набор интерфейсов, предоставляемых браузером JavaScript-коду для взаимодействия с браузером, операционной системой и устройством пользователя

Доступны через глобальные объекты

  • window
  • navigator
  • document

Не часть языка

Browser APIs не являются частью JavaScript — они предоставляются браузерным окружением

Что можно делать

  • Хранить данные на клиенте
  • Работать с устройством и сетью
  • Управлять окном и навигацией
  • Читать файлы и работать с медиа

Хранение данных в браузере

Механизмы хранения

localStorage

~5–10 МБ, хранится постоянно, только текущий origin

sessionStorage

~5–10 МБ, до закрытия вкладки, только текущая вкладка

Cookies

~4 КБ, время жизни задаётся явно, отправляются на сервер

IndexedDB

Сотни МБ, хранится постоянно, только текущий origin

localStorage

объект хранилища, данные из которого сохраняются бессрочно — до явного удаления кодом или пользователем

Данные привязаны к origin

протокол + домен + порт

Методы localStorage

  • localStorage.setItem(key, value) — сохранить значение
  • localStorage.getItem(key) — получить значение (или null )
  • localStorage.removeItem(key) — удалить ключ
  • localStorage.clear() — очистить всё хранилище
  • localStorage.length — количество ключей 

// Сохранение строки
localStorage.setItem('username', 'alice');

// Чтение
const name = localStorage.getItem('username');
console.log(name); // 'alice'

// Удаление
localStorage.removeItem('username');

// Полная очистка
localStorage.clear();

sessionStorage

аналог localStorage, но данные живут только в рамках одной вкладки и удаляются при её закрытии

Когда использовать

  • Временное сохранение состояния многошаговых форм
  • Данные, которые не должны "утекать" в другие вкладки
  • Кэш данных на время одного сеанса просмотра

// Сохраняем данные формы на время сессии
sessionStorage.setItem('draftMessage', 'Привет, это черновик...');

// Читаем при возврате на страницу
const draft = sessionStorage.getItem('draftMessage');
if (draft) {
  document.getElementById('messageInput').value = draft;
}

Сериализация

Web Storage хранит только строки — объекты нужно сериализовать перед записью и десериализовать при чтении

JSON.stringify(value)

преобразует JavaScript-значение в JSON-строку

JSON.parse(string)

разбирает JSON-строку обратно в JavaScript-значение 


const user = {
  id: 42,
  name: 'Alice',
  preferences: { theme: 'dark', language: 'ru' }
};

// Сериализация и сохранение
localStorage.setItem('user', JSON.stringify(user));

// Чтение и десериализация
const raw = localStorage.getItem('user');
const restoredUser = JSON.parse(raw);

console.log(restoredUser.preferences.theme); // 'dark'

Безопасное чтение


function getFromStorage(key) {
  try {
    const raw = localStorage.getItem(key);
    return raw ? JSON.parse(raw) : null;
  } catch (e) {
    console.error('Ошибка чтения:', e);
    return null;
  }
}

function saveToStorage(key, value) {
  try {
    localStorage.setItem(key, JSON.stringify(value));
  } catch (e) {
    console.error('Ошибка записи:', e);
  }
}

Cookies

небольшой фрагмент данных, который браузер хранит и автоматически отправляет на сервер при каждом HTTP-запросе — главное отличие от localStorage

Важные атрибуты

  • max-age / expires — время жизни
  • HttpOnly — недоступен через JS
  • Secure — только по HTTPS
  • SameSite — защита от CSRF-атак 

// Установка cookie
document.cookie = 'theme=dark; path=/; max-age=86400; SameSite=Strict';

// Чтение (строка, требует парсинга)
console.log(document.cookie); // 'theme=dark; user=alice'
Для удобной работы рекомендуется использовать современный Cookie Store API или библиотеки вроде js-cookie

Navigator API

Работа с устройством и окружением

Объект navigator

глобальный объект, предоставляющий информацию о браузере и устройстве пользователя

Доступен как window.navigator или просто navigator

navigator.userAgent

строка, идентифицирующая браузер и операционную систему 


console.log(navigator.userAgent);
// 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7) ...'

// Определение мобильного устройства
const isMobile = /Mobi|Android/i.test(navigator.userAgent);
console.log(isMobile); // true или false
userAgent легко подделать — предпочитайте feature detection

navigator.onLine

булево свойство, отражающее наличие сетевого подключения 


console.log(navigator.onLine); // true или false

window.addEventListener('online', () => {
  showNotification('Вы снова онлайн', 'success');
});

window.addEventListener('offline', () => {
  showNotification('Нет подключения к интернету', 'warning');
});

Geolocation API

позволяет получить географическое положение устройства. Требует явного разрешения пользователя

Ключевые методы

  • getCurrentPosition() — одноразовый запрос позиции
  • watchPosition() — непрерывное отслеживание
  • clearWatch(id) — прекратить отслеживание 

navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude, accuracy } = position.coords;
    console.log(`Широта: ${latitude}`);
    console.log(`Долгота: ${longitude}`);
    console.log(`Точность: ±${accuracy} м`);
  },
  (error) => {
    switch (error.code) {
      case error.PERMISSION_DENIED:
        console.error('Пользователь запретил доступ');
        break;
      case error.TIMEOUT:
        console.error('Превышено время ожидания');
        break;
    }
  },
  { enableHighAccuracy: true, timeout: 5000 }
);

Язык пользователя


// Основной язык браузера
console.log(navigator.language); // 'ru-RU', 'en-US'

// Все предпочтительные языки
console.log(navigator.languages); // ['ru-RU', 'ru', 'en']

// Автоопределение языка
function detectLanguage() {
  const preferred = navigator.language || navigator.languages[0];
  const lang = preferred.split('-')[0]; // 'ru' из 'ru-RU'
  return ['ru', 'en', 'de'].includes(lang) ? lang : 'en';
}

Работа с окном и вкладками

Объект window

глобальный объект браузерного окружения. Все глобальные переменные и большинство Browser APIs являются его свойствами

Location API

объект, представляющий текущий URL страницы и предоставляющий методы для навигации 


console.log(location.href);     // 'https://example.com/page?q=test#section'
console.log(location.protocol); // 'https:'
console.log(location.pathname); // '/page'
console.log(location.search);   // '?q=test'
console.log(location.hash);     // '#section'

// Парсинг строки запроса
const params = new URLSearchParams(location.search);
console.log(params.get('q')); // 'test'

Навигация через location


// Переход (записывается в history)
location.href = 'https://google.com';

// Переход (НЕ записывается в history)
location.replace('https://google.com');

// Перезагрузка страницы
location.reload();

History API

управление историей браузера — позволяет добавлять и изменять записи без перезагрузки страницы

Ключевая основа для SPA — Single Page Applications 

// Переходы по истории
history.back();    // как кнопка «Назад»
history.forward(); // как кнопка «Вперёд»
history.go(-2);    // назад на 2 шага

// Добавление записи без перезагрузки
history.pushState({ page: 'about' }, '', '/about');
console.log(location.pathname); // '/about' (без перезагрузки!)

// Обработка кнопки «Назад» в SPA
window.addEventListener('popstate', (event) => {
  renderPage(event.state?.page || 'home');
});

// Текущая позиция

setTimeout / setInterval

setTimeout

выполняет функцию один раз после задержки

setInterval

выполняет функцию многократно с заданным интервалом 


// setTimeout — однократное выполнение
const timerId = setTimeout(() => {
  console.log('Выполняется через 2 секунды');
}, 2000);

clearTimeout(timerId); // Отмена до срабатывания

// setInterval — периодическое выполнение
let count = 0;
const intervalId = setInterval(() => {
  count++;
  if (count >= 5) {
    clearInterval(intervalId);
  }
}, 1000);

File API

Работа с мультимедиа и файлами

input type="file"

HTML-элемент, открывающий системный диалог выбора файла 


<!-- Простой выбор одного файла -->
<input type="file" id="fileInput">

<!-- Выбор нескольких файлов -->
<input type="file" id="multipleInput" multiple>

<!-- Только изображения -->
<input type="file" id="imageInput" accept="image/*">
        

const input = document.getElementById('fileInput');

input.addEventListener('change', (event) => {
  const files = event.target.files; // FileList

  for (const file of files) {
    console.log(`Имя: ${file.name}`);
    console.log(`Тип: ${file.type}`);
    console.log(`Размер: ${(file.size / 1024).toFixed(2)} КБ`);
    console.log(`Изменён: ${new Date(file.lastModified).toLocaleString()}`);
  }
});

Интерфейсы File API

  • File — представляет файл (расширяет Blob )
  • FileList — список файлов из <input> или drag & drop
  • FileReader — читает содержимое файлов асинхронно
  • Blob — бинарный объект данных 

// Чтение файла как текст
function readFileAsText(file) {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = (e) => resolve(e.target.result);
    reader.onerror = () => reject(reader.error);
    reader.readAsText(file, 'UTF-8');
  });
}

// Использование
input.addEventListener('change', async (e) => {
  const file = e.target.files[0];
  if (file.type.startsWith('text/')) {
    const content = await readFileAsText(file);
    console.log('Содержимое:', content);
  }
});

Preview изображений


<input type="file" id="imageInput" accept="image/*">
<img id="preview" src="" alt="Preview"
     style="max-width: 300px; display: none;">
        

imageInput.addEventListener('change', (event) => {
  const file = event.target.files[0];

  // URL.createObjectURL — эффективнее, без копирования данных
  const objectUrl = URL.createObjectURL(file);
  preview.src = objectUrl;
  preview.style.display = 'block';

  // Освобождаем память после использования
  preview.onload = () => URL.revokeObjectURL(objectUrl);
});

Drag & Drop

Ключевые события

  • dragenter — элемент перетащили в зону
  • dragover — элемент над зоной (нужен preventDefault() )
  • dragleave — элемент покинул зону
  • drop — элемент был отпущен в зоне 

dropZone.addEventListener('dragover', (e) => {
  e.preventDefault();
  dropZone.classList.add('drop-zone--active');
});

dropZone.addEventListener('dragleave', () => {
  dropZone.classList.remove('drop-zone--active');
});

dropZone.addEventListener('drop', (e) => {
  e.preventDefault();
  dropZone.classList.remove('drop-zone--active');

  const files = e.dataTransfer.files;
  for (const file of files) {
    processFile(file);
  }
});

Дополнительные API

Clipboard API

предоставляет доступ к буферу обмена системы. Асинхронный, для чтения требует разрешения пользователя

Ключевые методы

  • navigator.clipboard.writeText(text) — записать текст
  • navigator.clipboard.readText() — прочитать текст
  • navigator.clipboard.write(items) — произвольные данные 

async function copyToClipboard(text) {
  try {
    await navigator.clipboard.writeText(text);
    console.log('Текст скопирован!');
  } catch (err) {
    console.error('Ошибка копирования:', err);
  }
}

// Кнопка «Копировать код»
document.querySelectorAll('.code-block').forEach((block) => {
  const button = block.querySelector('.copy-btn');
  button.addEventListener('click', async () => {
    await copyToClipboard(block.querySelector('code').textContent);
    button.textContent = '✓ Скопировано!';
    setTimeout(() => { button.textContent = 'Копировать'; }, 2000);
  });
});

Intersection Observer

эффективный способ отслеживать, находится ли элемент в зоне видимости — альтернатива отслеживанию события scroll

Основные применения

  • Ленивая загрузка изображений
  • Бесконечный скролл
  • Анимации при появлении элементов
  • Трекинг видимости (аналитика)

const observer = new IntersectionObserver(callback, {
  root: null,         // null = viewport
  rootMargin: '0px',  // отступы (как CSS margin)
  threshold: 0        // 0 = 1px видно, 1 = полностью виден
});

Ленивая загрузка


<img data-src="large-image.jpg" src="placeholder.jpg" class="lazy">
        

const imageObserver = new IntersectionObserver((entries, observer) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.remove('lazy');
      observer.unobserve(img); // Прекращаем наблюдение
    }
  });
}, { rootMargin: '200px 0px' }); // Загружаем за 200px до появления

document.querySelectorAll('img.lazy')
  .forEach((img) => imageObserver.observe(img));

Анимация при появлении


.fade-in {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}
.fade-in.visible {
  opacity: 1;
  transform: translateY(0);
}
        

const animationObserver = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
      animationObserver.unobserve(entry.target);
    }
  });
}, { threshold: 0.1 });

Web Storage Events

событие storage срабатывает в других вкладках того же origin при изменении localStorage

Событие не срабатывает в той вкладке, которая изменила данные — только в остальных 

// Синхронизация авторизации между вкладками
window.addEventListener('storage', (event) => {
  if (event.key === 'authToken') {
    if (event.newValue === null) {
      // Пользователь вышел в другой вкладке
      window.location.href = '/login';
    } else {
      updateUserUI(JSON.parse(event.newValue));
    }
  }
});

// Современный подход: BroadcastChannel API
const channel = new BroadcastChannel('app-sync');
channel.postMessage({ type: 'USER_LOGOUT' });
channel.addEventListener('message', (event) => {
  if (event.data.type === 'USER_LOGOUT') {
    window.location.href = '/login';
  }
});

Fullscreen API

позволяет переводить элементы страницы в полноэкранный режим

Ключевые методы и свойства

  • element.requestFullscreen() — запросить полноэкранный режим
  • document.exitFullscreen() — выйти из режима
  • document.fullscreenElement — текущий элемент или null
  • Событие fullscreenchange — при изменении режима 

async function toggleFullscreen(element) {
  if (document.fullscreenElement) {
    await document.exitFullscreen();
  } else {
    await element.requestFullscreen();
  }
}

fullscreenBtn.addEventListener('click', () => {
  toggleFullscreen(videoContainer);
});

document.addEventListener('fullscreenchange', () => {
  const isFullscreen = !!document.fullscreenElement;
  fullscreenBtn.textContent = isFullscreen ? '⛶ Выйти' : '⛶ На весь экран';
});
